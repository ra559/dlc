Virtualization

Virtualization is the process of running a virtual instance of a computer system in a layer abstracted from the actual hardware. Most commonly, it refers to running multiple operating systems on a computer system simultaneously.
-Opensource.com

Virtualization allows system administrators to replicate make-believe computers inside a single computer. The abstraction of computing resources can be achieved via software, hardware, or a mix of both. The concept of virtualization is believed to have been originated during the 1960s and 70’s when IBM started experimenting in time-sharing computing. Today, the computers that power the IT infrastructure of many organizations, including Cloud Computing, utilize virtualization techniques to create large aggregated pools of logical resources (CPUs, memory, disks, etc) from where an array of servers/services can run.

Virtualization is achieved via the use of a hypervisor. A hypervisor is the piece that allows the creation and management of virtual machines. A hypervisor can be physical (bare metal) or software. A virtual machine (VM) is the emulated equivalent of a computer system that runs on top of another system. There are two types of hypervisors, Type 1 and Type 2.

  • A Type 1 hypervisor installs on a computer before any operating system, also called a bare-metal hypervisor, and then it partitions the hardware for multiple VMs. This allows for the installation of different operating systems in each virtual machine. There are many vendors of type 1 hypervisors; two of the biggest being VMWare with ESXi and Citrix with XenServer. A popular usage of type 1 hypervisors is in server-hosted desktops where a server provides virtual desktops or VMs to multiple users over the local network or the cloud.
  • A Type 2 hypervisor installs in a host operating system as an application, also, it is known as a hosted hypervisor. Examples of Type 2 hypervisors include Oracle Virtualbox, VMware Workstation or Player, and Windows Client Hyper-V. A Type 2 hypervisor is not as powerful as a Type 1 hypervisor because it is dependent on the host OS to allot its computing power. A VM in a Type 2 hypervisor is not as secure or as fast as a VM in a Type 1 hypervisor. The advantage that Type 2 has over Type 1 is that it requires less specific hardware. For instance, most modem consumer-grade computers can use type 2 hypervisors.

Advantages of Virtualization

  • Smaller carbon footprint
  • Reduce hardware maintenance and acquisition cost
  • Increased application availability
  • Better cross-platform support
  • Provides testing and debugging environment

Setting up client-side virtualization

Any workstation that will implement virtual machines must meet the following criteria:

  • Sufficient CPU cores: Each VM has its own virtual processor, therefore, the host’s processor must be a multi-core processor with support for hardware-assisted virtualization (HAV), and Intel-V or AMD-V which is a technology that enhances the processor support for virtual machines.
  • Motherboard support: Most of today’s motherboards support HAV, Intel-V or AMD-V but many computer manufacturers disable this feature by default. This can always be enabled in BIOS/UEFI settings.
  • Enough RAM: It is crucial to have sufficient RAM since not only will the virtual machines consume RAM but so will the host operating system (in the case of type 2 hypervisors). Some hypervisors are designed so that each VM that is running ties up all the RAM that it has been assigned.
  • Storage space: Each VM has its own virtual hard drive (VHD) therefore it will tie up space from the physical hard drive.
  • Network connection: If the virtual machines will be used to connect to any network, make sure that the host has a good network card. It is possible to use multiple network cards.

When deciding how to use the overall budget for a virtualization workstation, prioritize the number of CPU cores and the amount of installed RAM.

  4. In Windows and Mac, VirtualBox can be installed the same way any other application is installed. In Linux systems, VirtualBox can be installed from the distribution repositories or by downloading the package from the download page.
