Containerization vs Virtualization

The choice of virtualization versus containerization depends on the business development and operational model, and the way applications are written and produced.
-Liquidweb.com

Even though containers and VMs appear to overlap in functionality, definition, and use cases, they are their own different set of breeds. (Below are similarities and comparisons)

# Virtualization Containerization
1 Includes an entire operating system. Larger system resource footprint and requirements.
2 Does not need to utilize entire operating system to function The host OS and the app guest container have to run similar OSs.
3 Each VM often needs its own kernel. Need a hypervisor. Slower creation and startup times. Minimal system resource footprint and requirements
4 Host dependent. The host OS can be completely different from the guest. Meaning you can run Windows, OS X, and Linux guest VMs inside a Linux host if wanted to. Each VM often needs its own kernel. Need a hypervisor. Slower creation and startup times.
5 More secure out of the box May be less secure out of the box, partly because of the easy availability of third-party images
6 Host dependent. The host OS can be completely different from the guest. Meaning you can run Windows, OS X, and Linux guest VMs inside a Linux host if wanted to. Fast creation and startup times
